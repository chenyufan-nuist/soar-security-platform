# 🛡️ 中小企业安全事件自动化响应平台（SOAR）设计文档

## 一、项目概述

### 1.1 项目背景

中小企业通常缺乏专业安全运营团队，面对常见安全事件（如钓鱼邮件、勒索软件告警）时响应效率低、流程混乱。

本项目旨在构建一个轻量级 SOAR 平台，实现：

- 安全事件自动化响应
- 标准化处置流程
- 降低人工参与成本

---

### 1.2 项目目标

- 实现安全事件的自动化剧本编排与执行
- 实现告警聚合与分类
- 提供工单驱动的安全运营流程
- 自动生成处置报告

---

### 1.3 核心能力

- 自动化响应（Playbook）
- 事件驱动架构（消息队列）
- 威胁情报联动（MISP）
- 工单闭环管理

---

## 三、核心模块设计

### 3.1 告警接入模块

#### 功能

- 接收外部告警（Webhook / API）
- 标准化告警格式
- 基础字段校验与签名校验（可选）

#### 输入示例

```json
{
  "type": "phishing",
  "source": "email_gateway",
  "user": "test@company.com",
  "ioc": "http://malicious.com",
  "timestamp": "2026-04-22T10:00:00"
}
```

---

### 3.2 告警聚合模块

#### 功能

- 去重（基于 IOC + 时间窗口）
- 分类（勒索 / 钓鱼 / 恶意 IP）
- 相似事件合并与优先级提升

#### 示例逻辑

```python
if alert.ioc in cache and within_5_minutes:
    merge_alert()
```

---

### 3.3 剧本引擎（Playbook Engine）

#### 核心思想

使用 JSON 或 YAML 定义自动化剧本，以步骤编排驱动执行。

#### 示例 Playbook（钓鱼事件）

```yaml
name: phishing_response
steps:
  - action: query_misp
    params:
      ioc: "{{ioc}}"

  - action: block_url
    params:
      url: "{{ioc}}"

  - action: notify_user
    params:
      user: "{{user}}"

  - action: create_ticket
    params:
      title: "钓鱼攻击响应"
```

---

### 3.4 自动响应模块

#### 支持动作

| 动作 | 描述 |
| ---- | ---- |
| block_ip | 封禁 IP |
| block_url | 封禁 URL |
| isolate_host | 隔离主机 |
| disable_account | 禁用账号 |

---

### 3.5 威胁情报模块（MISP）

#### 功能

- 查询 IOC 情报
- 判断风险等级
- 返回标签与建议处置策略

#### 示例代码

```python
def query_misp(ioc):
    response = requests.get(f"http://misp/api/search/{ioc}")
    return response.json()
```

---

### 3.6 工单系统

#### 功能

- 自动生成工单
- 跟踪处理状态
- 记录审计日志（谁在何时执行了什么动作）

#### 状态流转

```text
OPEN -> IN_PROGRESS -> RESOLVED -> CLOSED
```

---

### 3.7 报告生成模块

#### 输出示例

```json
{
  "event": "phishing",
  "actions": [
    "查询威胁情报",
    "封禁 URL",
    "通知用户"
  ],
  "status": "resolved"
}
```

### 4.1 告警表（alerts）

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |

## 五、接口设计（API）

### 5.1 接收告警

```http
POST /api/alert
```

请求体：标准化告警 JSON。

---

### 5.2 查询工单

```http
GET /api/tickets
```

支持分页参数：page、size。

---

### 5.3 手动触发剧本

```http
POST /api/playbook/run
```

请求体：事件 ID + 剧本名称。

## 六、消息队列设计（RabbitMQ）

### 队列划分

| 队列 | 作用 |
| ---- | ---- |
| alert_queue | 告警输入 |
| playbook_queue | 剧本执行 |
| action_queue | 自动响应 |
| report_queue | 报告生成 |
