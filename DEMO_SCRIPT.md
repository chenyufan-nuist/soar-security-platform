# 🎬 SOAR 平台演示视频录制脚本

这是一份为您专门定制的、用于录制演示视频的详细分步清单。您可以按照以下步骤一边解说一边操作录屏。

## 准备阶段 (视频录制前)
1. 确保所有进程已停止（清除之前可能残留的终端）。
2. 打开 VS Code，确保 `d:\信息作品竞赛\` 在您的工作区中。

---

## 🟢 第一幕：系统启动 (0:00 - 0:30)

**解说词参考**：“大家好，现在为大家演示中小企业安全事件自动化响应（SOAR）平台。首先我们一键启动整个系统...”

**操作步骤**：
1. 在 VS Code 的终端中运行一键启动脚本：
   ```bash
   .\start.bat
   ```
2. 等待两个外置黑色 CMD 窗口弹出并显示“Started server process”以及“VITE ready”。
3. 打开浏览器，访问前端地址：[http://localhost:5173](http://localhost:5173)。
4. 页面成功加载，展现赛博朋克风格的深色控制台。

---

## 🟢 第二幕：全局概览 (0:30 - 1:00)

**解说词参考**：“进入平台后，我们首先看到的是全局视图（GLOBAL VIEW），这里大屏直观展示了当前系统的安全态势，包括待处理告警数量、已解决的事件以及各类安全威胁的占比统计。”

**操作步骤**：
1. 鼠标在 **GLOBAL VIEW (全局视图)** 停留，光标指引右上角的 `SYSTEM ONLINE` 状态。
2. 简略滑动查看仪表盘的数据。

---

## 🟢 第三幕：模拟安全威胁接入 (1:00 - 1:30)

**解说词参考**：“接下来，我们模拟一次真实的黑客钓鱼攻击产生告警。大家请看，目前告警中心是常规状态。我们通过外部流量网关推送一条钓鱼告警进入系统。”

**操作步骤**：
1. 在 VS Code 内置终端（或者另外开一个终端），运行模拟脚本（此脚本会向后台发送真实的告警数据）：
   ```bash
   python demo.py
   ```
2. 马上切回浏览器，点击上方导航栏的 **Threat Alerts (威胁告警)**。
3. 指出列表中刚刚新增的一条红色或高亮的 `phishing`（钓鱼攻击）告警，强调其源自 `http://fake-login-portal.com`。

---

## 🟢 第四幕：自动化编排与响应（核心高光） (1:30 - 2:30)

**解说词参考**：“面对新产生的钓鱼威胁，传统的安全人员需要手动去封禁IP并联系员工。但在我们的 SOAR 平台，这一切都可以通过剧本自动化完成。现在我们进入自动化编排引擎。”

**操作步骤**：
1. 点击导航栏切换到 **Automation Engine (自动化编排引擎)**。
2. 在左侧的【Target Alert ID】下拉框中，选择刚刚那条新的钓鱼告警。
3. 在【Playbook Module】下拉框中，选择 `phishing_response` 剧本。
4. 解说：“右侧是我们的剧本库，选中了应对钓鱼事件的自动化剧本后，我们点击执行。”
5. 点击 **AUTHORIZE DEPLOYMENT** 按钮。
6. **弹窗高光时刻**：屏幕中央会弹出【⚡ 执行结果 (EXECUTION REPORT)】。
7. 鼠标逐一划过弹窗里的执行步骤，解说：
   - “剧本首先接通 MISP 威胁情报库验证了该恶意链接。”
   - “随后自动调用防火墙接口拦截了该恶意 URL。”
   - “接着自动通过企微/邮件提醒了受害员工。”
   - “最后，平台自动生成了一份处置工单。”
8. 点击弹窗右下角的 **ACKNOWLEDGE & CLOSE**。

---

## 🟢 第五幕：工单闭环与总结 (2:30 - 3:00)

**解说词参考**：“自动化剧本执行完毕后，所有证据链和操作记录都已经汇总到了工单中心。我们来看一下。”

**操作步骤**：
1. 点击导航栏切换到 **Dispatch Center (工单调度中心)**。
2. 页面中会出现刚刚剧本自动生成的新工单：`[PHISHING] 自动响应处置`（或相关标题）。
3. 展开工单详情，展示它已经自动记录了所有剧本刚才的操作明细和结案报告。
4. **结束语**：“到此，一次本来需要十几分钟的人工响应流程，在 SOAR 平台上不到半分钟就实现了全自动闭环。演示完毕，谢谢大家！”

---
## 💡 视频录制的小建议：
- 推荐使用全屏录制或者浏览器F11全屏模式，视觉效果更沉浸。
- 执行剧本弹出窗口时，语速稍微放慢一点，让评委能看清弹窗里的每一步 `SUCCESS` 操作。
